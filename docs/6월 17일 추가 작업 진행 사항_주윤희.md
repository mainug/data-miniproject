### 1. 데이터 수집 확장

- **다일(multi-day) 박스오피스 수집** — 현재 어제 하루만 수집하는데, 7일/30일치를 모아야 추세 분석 가능
- **TMDB 추가 엔드포인트** — `now_playing`, `top_rated`, `trending` 등으로 데이터 풀 확대
- **영화 상세정보 보강** — 감독/출연진(credits), 수익(revenue), 제작비(budget) 등

### 2. 분석 스크립트 (프로젝트 핵심 기능들)

- **흥행 vs 평점 상관관계** — scatter plot + 상관계수 (프론트에 `RatingScatter.tsx` 컴포넌트가 이미 있음)
- **연도별 트렌드** — 개봉연도별 평점/수익 추이 (`YearTrendChart.tsx` 대응)
- **장르별 심화 분석** — 장르별 평균 평점/수익 비교

### 3. 프론트용 chart_data.json 확장

현재 2종(박스오피스, 장르분포)만 있는데, 프론트 차트 컴포넌트가 7종이나 준비되어 있습니다. 나머지 5종의 데이터도 만들어두면 프론트에서 바로 연결 가능합니다.

- **파일**: kobis_boxoffice_30d.csv, .json
- **300건** (30일 × 10편/일), 기간: 2026-05-18 ~ 2026-06-16
- 필드: date, rank, movieCd, movieNm, openDt, salesAmt, salesShare, audiCnt, audiAcc, scrnCnt, showCnt

- **파일**: tmdb_movies.csv, .json
- **231편** (중복 제거 후)
- 소스: popular(99) + now_playing(22) + top_rated(92) + trending(18)

### TMDB 상세정보 보강

- **파일**: movies_enriched.csv, .json
- **231편**, 추가 컬럼: runtime, budget, revenue, director, cast_top3, collection_name, production_countries, tagline, status

### 수집 스크립트

    - `-skip-kobis`, `-skip-tmdb`, `-skip-details` 옵션으로 선택적 실행

- **상관분석 결과:**
    - 평점 ↔ 수익: r = **0.1725** (약한 상관 — 평점 높다고 흥행하는 건 아님)

    - 제작비 ↔ 수익: r = **0.7451** (강한 상관 — 돈 많이 쓸수록 많이 번다)

## KOBIS / TMDB 2탭 구조

### TMDB 페이지 (`🎬 TMDB`)

상단부터 순서대로:

1. **통계 배너** — 총 영화수, 평균 평점, 글로벌 수익 합계, 수집 소스 4종
2. **필터바** — 연도 범위, 정렬 기준, Top N, 장르 필터
3. **영화 랭킹** — 포스터 카드 그리드
4. **장르 분석** — 장르 차트 + 제작국가별
5. **트렌드 분석** — 연도별 트렌드 + 감독별 평점
6. **흥행 분석** — 상관관계 scatter + ROI + Top N

### KOBIS 페이지 (`🇰🇷 KOBIS`)

1. **통계 배너** — 수집 기간, 총 관객수, 총 매출, 등장 영화 수
2. **30일 추세** — 일별 관객수/매출 라인차트
3. **일별 상세** — 날짜 선택 → 박스오피스 테이블 + 당일 관객 분포

## —> 수정 내용

### 1. 데이터 중복 표시 해결

- **원인**: `useBoxOffice`가 30일치 300건을 필터링 없이 전부 반환
- **수정**: useBoxOffice.ts — 선택된 날짜(또는 최신 날짜)로 필터링 후 rank순 정렬 → **10건만 표시**

### 2. 날짜 선택 시 테이블 + 차트 연동

- 날짜 `<select>` 드롭다운으로 교체 (수집된 30일 목록에서 선택)
- 선택하면 해당 날짜의 박스오피스 테이블 + 당일 관객수 분포 차트가 함께 갱신

### 3. 숫자 포맷 개선

- 당일 관객 1만 미만 → `1,320명`, 1만 이상 → `3.2만`
- 매출 1억 미만 → `만원 단위`, 1억 이상 → `3.2억`
