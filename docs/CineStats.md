🎬 CineStats 프로젝트 진행 상황 요약 (Data Pipeline 중심)
1. 📊 데이터 수집 및 가공 (Data Analytics)
오픈 API(KOBIS, TMDB)를 활용하여 대시보드 및 추천 기능에 필요한 데이터를 수집하고, 백엔드/프론트엔드에서 사용하기 가장 최적화된 형태로 전처리를 완료했습니다.

일별 박스오피스 수집 (fetch_kobis)
매일 영화진흥위원회(KOBIS) API를 호출하여 전일 기준 Top 10 순위, 관객수, 매출액 등을 수집합니다.
시계열 데이터 관리를 위해 집계 기준일(date) 필드를 가공하여 명시적으로 추가했습니다.
영화 메타데이터 수집 (fetch_tmdb, tmdb_transform.py)
TMDB API를 활용해 인기 영화의 기본 정보(제목, 평점, 포스터 URL 등)를 수집합니다.
프론트엔드의 시각화(상영시간 차트) 및 추천 기능을 위해 상세 API를 추가 호출하여 상영 시간(runtime) 과 시리즈/세계관(collection_name) 데이터를 확보했습니다.
[핵심 가공] 복잡한 장르 ID를 데이터 파이프라인 단에서 직접 한글 문자열 배열(예: ["공포", "액션"])로 매핑 및 변환하여, 백엔드와 프론트엔드의 연산 부담을 크게 줄였습니다.
관련 콘텐츠 추천 데이터 구축 (fetch_similar_movies.py)
영화별로 TMDB의 '비슷한 영화' API를 호출하여, [{"movie_id": 123, "similar_movie_ids": [456, 789]}] 형태의 추천용 매핑 데이터셋(JSON)을 구축했습니다.
2. 🤝 팀 협업 및 명세서 확립 (DATA_SPEC.md)
백엔드 담당자와 소통하여 데이터 스펙 및 API 통신 방식을 확립하고 문서화를 완료했습니다.

식별자(PK) 및 중복 방지 정책: 박스오피스 테이블의 PK는 JPA 관리에 용이한 자동 증가 Long ID를 사용하되, (movieCd, date) 조합에 Unique 제약조건을 걸어 UPSERT(덮어쓰기) 처리하기로 합의했습니다.
최적화(CQRS 패턴 도입): 리소스 절약을 위해 영화 목록 조회 시 수익(revenue) 필드는 0으로 넘기고, 상세 페이지 조회 시점에 별도로 값을 채우기로 결정했습니다.
Bulk API Push 방식 채택: 데이터 파이프라인에서 전처리가 끝난 데이터를 백엔드의 벌크 등록용 REST API(POST /api/movies/bulk 등)로 직접 밀어 넣어주는(Push) 결합도 낮은 아키텍처를 설계했습니다.
3. 🚀 파이프라인 자동화 및 모니터링
데이터 수집부터 백엔드 DB 적재까지의 전 과정을 자동화하고 테스트 환경을 구축했습니다.

백엔드 자동 전송 스크립트 (push_to_backend.py)
pandas DataFrame으로 가공된 데이터를 백엔드 Bulk API 요구 스펙(JSON 배열)에 맞춰 변환 후 requests.post를 통해 자동 전송합니다.
Slack 오류/성공 알림 연동
전송 과정에서 네트워크 오류나 HTTP 상태 코드 에러(4xx, 5xx)가 발생하면, 즉각적으로 Slack Webhook을 통해 에러 로그와 상태 코드를 알림으로 전송하도록 구성했습니다. 성공 시에도 완료 알림이 전송됩니다.
Windows 작업 스케줄러 (run_pipeline.bat)
작업 디렉터리 꼬임 현상을 방지하는 배치 파일을 작성하고, 매일 새벽에 백그라운드에서 스크립트가 자동 실행되도록 스케줄러 세팅을 완료했습니다.
로컬 테스트 환경 (test_dataframe.py, test_api.py)
백엔드 서버가 닫혀있을 때도 DA 단독으로 데이터프레임 형태를 점검하거나 API 키 정상 작동 여부를 터미널에서 즉시 검증할 수 있는 디버깅 툴을 마련했습니다.
4. 💻 프론트엔드 대시보드 기반 (Frontend)
React 기반 시각화 대시보드 (App.tsx 등)
수집된 데이터(또는 Mock 데이터)를 기반으로 Recharts를 활용하여 장르 분포 차트, 연도별 트렌드 차트, 평점 산점도 등을 구현해 두었습니다.
다크 모드 지원 및 애니메이션(Framer Motion)이 적용된 상태입니다.
💡 향후 진행 계획 (Next Steps)
백엔드 Bulk API 개설 대기: 백엔드 팀에서 POST /api/movies/bulk 등의 엔드포인트와 UPSERT 로직 구현이 완료되면, 자동화 파이프라인을 최종 연동 테스트합니다.
이상형 월드컵 데이터셋 가공: TMDB API를 활용해 인지도 높은 64개의 영화를 필터링하여 이상형 월드컵 후보풀(Pool) 데이터를 추가 생성할 예정입니다.
